## IAM Hands-On: Creating Users and Groups

### Accessing IAM

* In the **AWS Console**, type **IAM** in the search bar and open the **IAM Console**.
* On the top-right, notice:

  * The **region selector** is **disabled** — IAM is a **global service**.
  * IAM users and settings apply **across all regions**.

---

### Why Create IAM Users?

* Currently using the **root user**, which:

  * Has full permissions.
  * Should only be used for **initial setup**.
* Instead, create **IAM users** for safe daily operations (e.g., admin users).

---

### Creating an IAM User

1. Go to **Users → Create user**.
2. Enter a **username** (e.g., `Stefan`).
3. Choose **Access type**:

   * **Management Console access**
4. Choose **Authentication method**:

   * Use **IAM user** (simpler and exam-relevant)
   * Optionally, **set a custom password** for yourself.
   * Disable “Require password reset” if it’s your own user.

---

### Adding Permissions

* Option 1: Attach permissions directly.
* Option 2 (**recommended**): Assign the user to a **group**.

**Example:**

* Create a group named `admin`.
* Attach the policy **AdministratorAccess**.
* Add user `Stefan` to the `admin` group.

---

### Adding Tags (Optional)

* Tags are key–value metadata for AWS resources.
  Example:

  ```text
  Department = Engineering
  ```
* Tags help with organization, cost tracking, and automation.

---

### Review and Create

* Review details:

  * Username
  * Group and permissions
  * Tags
* Create the user.

Once created:

* You can **download a CSV** with login details.
* Or **email sign-in instructions** to the user.

---

### Viewing Users and Groups

* **Users section:** shows all IAM users (e.g., `Stefan`)
* **Groups section:** shows groups (e.g., `admin`)

  * Group `admin` contains `Stefan`
  * Group has the policy `AdministratorAccess`
* User `Stefan` **inherits** group permissions.

> ✅ Benefit: Managing permissions through groups simplifies administration.

---

### Customizing the Sign-In URL

* Each AWS account has a **sign-in URL**.
* You can create an **Account Alias** for easier login:

  ```
  Example: stefan-v5
  ```
* This alias becomes part of your sign-in URL.

---

### Logging In as an IAM User

**Options:**

* Log in via the sign-in URL using:

  * **Account ID** or **Alias**
  * **IAM Username**
  * **Password**
* For dual login:

  * Use **normal browser** for root account.
  * Use **private/incognito window** for IAM user.

> This allows you to operate both accounts side-by-side without logging out.

---

### Visual Indicators

| Account Type     | Top-Right Display     | Notes                           |
| ---------------- | --------------------- | ------------------------------- |
| **Root Account** | Only Account ID       | Full privileges                 |
| **IAM User**     | Account ID + Username | Limited to assigned permissions |

---

### Best Practices

* **Never lose** your root or IAM admin credentials.
* **Use IAM users** for daily tasks.
* Keep **root** for:

  * Billing setup
  * Account recovery
  * Rare administrative needs
* In this course:

  * You’ll mostly use **IAM user**
  * Instructor will specify if **root access** is needed

---

### Summary

* IAM is **global**, not region-specific.
* Create **users** for individual access.
* Group users (e.g., Admins) to assign permissions collectively.
* Use **AdministratorAccess** for your admin user.
* Create and use **Account Alias** for simpler login.
* Operate with **IAM users**, not the root account.
