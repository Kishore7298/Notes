## Accessing AWS: Console, CLI, and SDK

### 1. **Overview**

There are **three main ways to access AWS**:

1. **Management Console** – Web-based interface.
2. **Command Line Interface (CLI)** – Terminal-based interaction.
3. **Software Development Kit (SDK)** – Programmatic access from code.

---

### 2. **AWS Management Console**

* Accessed through a **web browser**.
* Protected by **username**, **password**, and possibly **MFA**.
* Best suited for **beginners** and **manual management** of AWS resources.

---

### 3. **AWS Command Line Interface (CLI)**

#### **Purpose**

* Enables AWS interaction via the **terminal/command prompt** using commands.
* Allows scripting, automation, and bulk resource management.

#### **Authentication**

* Protected by **Access Keys**:

  * **Access Key ID** → like a username.
  * **Secret Access Key** → like a password.
* Access keys are **private** and should **never be shared**.

#### **Generating Access Keys**

1. Log in to the **AWS Management Console**.
2. Go to your **IAM User settings**.
3. Click **“Create Access Keys”**.
4. Download the credentials immediately (you won’t see them again).

⚠️ **Security Reminder**

* Treat your keys as credentials:

  * Never share them.
  * Rotate regularly.
  * Delete unused keys.

#### **Example (Fake Keys)**

```
Access Key ID: AKIAEXAMPLE12345
Secret Access Key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

#### **CLI Features**

* Direct access to **AWS public APIs**.
* Commands usually start with the word `aws`, e.g.:

  ```bash
  aws s3 ls
  aws ec2 describe-instances
  ```
* **Open-source** (available on GitHub).
* Enables **automation** using shell scripts or CI/CD pipelines.
* Some advanced users manage AWS **exclusively** through the CLI.

---

### 4. **AWS SDK (Software Development Kit)**

#### **Purpose**

* Used to access AWS programmatically **within application code**.
* Ideal for developers integrating AWS into their apps.

#### **Key Points**

* **Language-specific** libraries for programmatic access to AWS APIs.
* Used **inside code**, not from the terminal.

#### **Supported Languages**

* **JavaScript (Node.js)**
* **Python (Boto3)**
* **Java**
* **Go**
* **C++**
* **.NET**
* **PHP**
* **Ruby**
* **Mobile SDKs:** Android, iOS
* **IoT SDK:** For Internet of Things devices

#### **Example**

* The **AWS CLI** itself is built on the **AWS SDK for Python (Boto3)**.

---

### 5. **Summary**

| Method                 | Use Case                      | Authentication             | Notes                        |
| ---------------------- | ----------------------------- | -------------------------- | ---------------------------- |
| **Management Console** | Web UI for AWS                | Username + Password + MFA  | Best for manual operations   |
| **CLI**                | Command-line automation       | Access Key ID + Secret Key | Great for scripting & DevOps |
| **SDK**                | Application-level integration | Access Key ID + Secret Key | Used inside your code        |
