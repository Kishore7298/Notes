## IAM Security: Password Policies & Multi-Factor Authentication (MFA)

### 1. **Why Protection Matters**

* IAM users and groups can be compromised if not properly secured.
* Two main defense mechanisms:

  1. **Password Policies**
  2. **Multi-Factor Authentication (MFA)**

---

### 2. **Password Policy**

#### **Purpose**

* To strengthen account security and reduce brute-force attack risk.

#### **Configurable Options**

| Option                          | Description                                                                                  |
| ------------------------------- | -------------------------------------------------------------------------------------------- |
| **Minimum Password Length**     | Define how long passwords must be.                                                           |
| **Character Requirements**      | Enforce inclusion of uppercase, lowercase, numbers, and special characters (e.g., `?`, `!`). |
| **Password Change Permissions** | Allow or restrict IAM users from changing their own passwords.                               |
| **Password Expiration**         | Require users to reset their passwords periodically (e.g., every 90 days).                   |
| **Prevent Password Reuse**      | Stop users from reusing recent passwords.                                                    |

#### **Benefits**

* Reduces risk of brute-force attacks.
* Improves overall account hygiene and compliance posture.

---

### 3. **Multi-Factor Authentication (MFA)**

#### **Purpose**

* Adds an extra layer of security **on top of passwords**.
* Protects users (especially administrators) from unauthorized access.

#### **Concept**

* **Something you know:** your password.
* **Something you have:** a physical or virtual security device.

#### **Example**

* **Alice** knows her password and has an MFA token generator.
* Even if her password is stolen, an attacker **cannot access the account** without the MFA device.

---

### 4. **MFA Device Types**

| Type                            | Description                               | Example                         | Notes                                                                        |
| ------------------------------- | ----------------------------------------- | ------------------------------- | ---------------------------------------------------------------------------- |
| **Virtual MFA Device**          | App-based authentication on a phone       | Google Authenticator, **Authy** | Supports multiple tokens per device (Authy supports multiple users/accounts) |
| **U2F Security Key**            | Physical hardware key                     | **YubiKey** by Yubico           | Works with multiple users on one key; third-party device                     |
| **Hardware Key Fob MFA Device** | Physical token with a display             | Provided by **Gemalto**         | Also third-party device                                                      |
| **GovCloud MFA Device**         | Specialized key fob for AWS GovCloud (US) | Provided by **SurePassID**      | Only for AWS GovCloud users                                                  |

---

### 5. **Key Takeaways**

* Always protect **root** and **admin IAM users** with MFA.
* **Password policies** enforce account-level discipline.
* **MFA** adds a strong second layer of security.
* AWS supports both **virtual** and **physical** MFA devices.
* Understanding MFA options is **important for the exam**.
