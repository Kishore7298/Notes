## What are Containers?
- Lightweight, isolated environments sharing the same OS kernel
- Unlike VMs, containers don't have their own OS kernel
- Examples: LXC, LXD, Docker uses LXC containers

## How Docker Works

### OS Structure:
- OS = Linux Kernel + Software (UI, Drivers, Tools, etc.)
- Containers use host OS kernel, and add only the software layer of different OS flavors

### Example:
- Host OS: Ubuntu
- Container: Can be Debian, Fedora, CentOS, etc. (all Linux-based)
- âŒ Cannot run Windows containers on Linux Docker host

## Containers vs. Virtual Machines (VMs)

| Aspect           | Containers (Docker)         | Virtual Machines            |
|-----------------|:---------------------------|:---------------------------|
| OS Kernel       | Shared with host           | Separate per VM           |
| Boot Time       | Seconds                    | Minutes                   |
| Resource Usage  | Lightweight (MBs)          | Heavy (GBs)              |
| Isolation Level | Moderate                   | High                     |
| Flexibility     | Same kernel only           | Different OS possible    |
| Use Case        | App packaging and shipping | Full OS virtualization   |
